# Курсовая работа: Автоматизированная информационная система "Летная школа"

## Описание проекта

Данный проект представляет собой курсовую работу по разработке моделей и диаграмм для автоматизированной информационной системы управления летной школой.

## Структура проекта

### Диаграммы

Все диаграммы находятся в директории `diagrams/` и разработаны с использованием PlantUML.

#### 1. IDEF0 Диаграммы (Функциональная модель)

**Путь:** `diagrams/idef0/`

IDEF0 - методология функционального моделирования, используемая для описания бизнес-процессов. Модель соответствует стандарту Р 50.1.028-2001.

**Созданы две версии моделей: автоматизированная и неавтоматизированная (по 6 диаграмм каждая)**

**Автоматизированная версия** (`diagrams/idef0/automated/`):
1. **A-0_context.puml** - Контекстная диаграмма с АИС
2. **A0_decomposition.puml** - Декомпозиция первого уровня (5 процессов)
3. **A1_admission.puml** - Прием и регистрация студентов
4. **A2_planning.puml** - Планирование обучения
5. **A3_lessons.puml** - Проведение занятий
6. **A4_control.puml** - Контроль успеваемости

**Неавтоматизированная версия** (`diagrams/idef0/non-automated/`):
1. **A-0_context.puml** - Контекстная диаграмма без АИС
2. **A0_decomposition.puml** - Декомпозиция первого уровня (5 процессов)
3. **A1_admission.puml** - Прием и регистрация студентов
4. **A2_planning.puml** - Планирование обучения
5. **A3_lessons.puml** - Проведение занятий
6. **A4_control.puml** - Контроль успеваемости

**Особенности:**
- Каждая диаграмма содержит максимум 4-5 элементов на каждой стороне (входы, выходы, управление, механизмы)
- Автоматизированная версия показывает использование АИС "Летная школа"
- Неавтоматизированная версия показывает традиционный бумажный документооборот

**Дополнительная документация:**

- **MODEL_TEXT_AND_GLOSSARY.md** - Текст модели и глоссарий
  - Подробное описание цели моделирования
  - Точка зрения
  - Описание всех процессов
  - Глоссарий терминов

#### 2. IDEF3 Диаграммы (Модель потоков работ)

**Путь:** `diagrams/idef3/`

IDEF3 - методология моделирования процессов, ориентированная на описание последовательности действий и временных аспектов.

**Созданные диаграммы:**

1. **flight_lesson_workflow.puml** - Процесс проведения полетного занятия
   - Последовательность действий: предполетная подготовка, инструктаж, осмотр самолета, полет, разбор, внесение данных
   - Точки принятия решений (безопасность, исправность)
   - Временные характеристики каждого этапа
   - Общая длительность: 3-5 часов

2. **student_registration_workflow.puml** - Процесс регистрации студента
   - Последовательность действий от подачи заявления до зачисления
   - Параллельные процессы (проверка документов и медосмотр)
   - Точки синхронизации
   - Временные характеристики: 1-2 рабочих дня

#### 3. DFD Диаграммы (Диаграммы потоков данных)

**Путь:** `diagrams/dfd/`

DFD (Data Flow Diagram) - диаграммы потоков данных в нотации Гейна-Сарсона.

**Созданные диаграммы:**

1. **dfd_gane_sarson.puml** - Диаграмма потоков данных
   - Внешние сущности: Студент, Инструктор, Администратор, Регулирующие органы
   - Процессы: Регистрация, Управление расписанием, Учет занятий, Контроль успеваемости, Формирование отчетов
   - Хранилища данных: Студенты, Инструкторы, Самолеты, Курсы, Занятия, Экзамены, Сертификаты
   - Потоки данных между всеми компонентами

#### 4. IDEF1X Модель (Информационная модель)

**Путь:** `MigrationService/1ver.ercmdl` (XML-файл для Ramus)

IDEF1X - методология моделирования данных для проектирования реляционных баз данных.

**Существующие сущности:**
- Instructor (Инструктор)
- Student (Студент)
- Aircraft (Самолет)
- Course (Курс)
- Lesson (Занятие)
- Exam (Экзамен)
- Certificate (Сертификат)
- StudentCertificate (Сертификаты студента)
- LessonStatusChange (Изменения статуса занятия)

**Добавленная сущность (новая):**
- **MaintenanceLog (Журнал технического обслуживания)**
  - ID_Записи (первичный ключ)
  - ID_Самолета (внешний ключ на Aircraft)
  - Дата_Обслуживания
  - Тип_Обслуживания
  - Описание_Работ
  - Стоимость
  - Механик
  - Примечания

Сущность имеет 8 атрибутов (требовалось минимум 5).

## Как использовать диаграммы

### Просмотр диаграмм PlantUML

1. **Онлайн:**
   - Откройте [PlantUML Online Editor](http://www.plantuml.com/plantuml/uml/)
   - Скопируйте содержимое .puml файла в редактор
   - Диаграмма отобразится автоматически

2. **VS Code:**
   - Установите расширение "PlantUML"
   - Откройте .puml файл
   - Нажмите Alt+D для предварительного просмотра

3. **Экспорт в изображения:**
   ```bash
   # Установите PlantUML
   # Затем выполните:
   java -jar plantuml.jar diagrams/**/*.puml
   ```

### Работа с IDEF1X моделью

1. Откройте файл `MigrationService/1ver.ercmdl` в Ramus Educational 2.0.1
2. Модель отобразится в графическом редакторе
3. Можно экспортировать в различные форматы

## Предметная область

**Летная школа** - образовательное учреждение, осуществляющее подготовку пилотов гражданской авиации.

### Основные процессы:

1. **Прием студентов** - регистрация кандидатов, проверка документов, медицинское освидетельствование
2. **Планирование** - формирование групп, составление расписания, распределение ресурсов
3. **Обучение** - теоретические занятия, работа на тренажерах, полетная практика
4. **Контроль** - тестирование, экзамены, оценка прогресса
5. **Сертификация** - выдача документов об образовании и летных сертификатов

### Ресурсы:

- **Студенты** - обучающиеся, проходящие курс подготовки пилотов
- **Инструкторы** - квалифицированные пилоты, осуществляющие обучение
- **Самолеты** - учебная авиационная техника
- **Курсы** - программы обучения (PPL, CPL, ATPL и др.)
- **Учебные классы** - помещения для теоретических занятий
- **Тренажеры** - симуляторы полета
